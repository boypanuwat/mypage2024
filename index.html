คุณสามารถดูคู่มือการติดตั้ง Appium และ Playwright หรือเรียนรู้เพิ่มเติมเกี่ยวกับ AI และ Prompt Engineering สำหรับงาน QA ได้

ภาพรวมทักษะและประสบการณ์

ในฐานะผู้เชี่ยวชาญด้าน QA ผมมีประสบการณ์และทักษะที่หลากหลายในการรับรองคุณภาพของซอฟต์แวร์ ทั้งในส่วนของ Web และ Mobile App Testing

สายงานและทักษะ

  • ทำงานด้าน QA (Quality Assurance) และเชี่ยวชาญใน Web และ Mobile App Testing
  • มีประสบการณ์ทั้ง Manual และ Automated Testing โดยเฉพาะเครื่องมือ:
    • Playwright (เชี่ยวชาญ, ใช้งานจริง, มีโครงสร้างที่ชัดเจน)
    • Appium (ใช้งาน JavaScript สำหรับ Mobile Testing)
    • SeleniumLibrary ใน Robot Framework (เคยใช้กับ Web)
    • Cypress (เคยใช้มาก่อน, สนใจเปลี่ยนมา Playwright)
  • ใช้งานเครื่องมือในการจัดการ Test Case และ Test Report: GitLab, Sourcetree, Jenkins, Allure Report, Google Sheets (Automate Jira Report), Jira + AIO Test

ลักษณะงานปัจจุบัน

  • ทำ QA ให้กับหลายโปรเจกต์ใน Urbanice App Ecosystem: Urbanice Resident App, Niti Urbanice App (นิติบุคคล), Living Mart (Marketplace ในแอป), URB Home Service Backoffice
  • วางเทสเคสละเอียด มีการเขียน Test Steps, Expected Result, Test Data แบบโครงสร้างครบถ้วน
  • กำลังสอนทีม QA ใหม่ (Junior QA) ให้ใช้ Playwright และ Appium

วิธีการทำงาน

  • ชอบเขียน Test Script ที่มีโครงสร้างและรูปแบบที่เป็นมาตรฐาน: ใช้ JSON เก็บ Test Data และ Locators, ใช้ getByRole, locator แบบ Clean Code, ใส่ Comment ภาษาไทยเพื่อให้ทีมเข้าใจง่าย
  • สนใจเรื่อง CI/CD Integration
  • พยายามลด Resource เช่น หลีกเลี่ยง Emulator, ใช้ Real Device แทนในการเทส Mobile

เป้าหมาย

  • ทำให้ทีม QA มี Productivity สูงขึ้น
  • ทำ Automation ที่ใช้งานง่ายและ Maintain ง่าย
  • ทำ Report ที่อ่านง่ายและเข้าใจง่ายสำหรับผู้บริหาร (ชอบ Google Sheets, Allure, Excel Exporter)

AI และ Prompt Engineering สำหรับงาน QA

AI-enabled Employee หรือ AI-powered Assistant คือการใช้ AI มาช่วย "ทำงานแทน" หรือ "ช่วยงาน" พนักงาน เพื่อเพิ่มประสิทธิภาพ ลดเวลา และช่วยให้โฟกัสกับงานที่สำคัญ

AI สำหรับ Test Automation

  • Self-healing Test Scripts: AI ช่วยแนะนำหรือแก้ไข Locator โดยอัตโนมัติเมื่อ UI เปลี่ยน
  • AI-based Test Generation: AI สร้าง Test Case หรือ Automation Script เบื้องต้นจาก Requirement
  • AI Visual Testing: ตรวจจับความผิดปกติของ UI ด้วยการเปรียบเทียบภาพโดย AI
  • Predictive Testing: แนะนำ Feature ที่ควรโฟกัส Regression Test โดยอิงจาก Impact, Frequency, และ Historical Bug Data

AI สำหรับ Test Management

  • AI-powered Reporting: สรุปผลการทดสอบอัตโนมัติ สร้าง Test Summary Report ที่เข้าใจง่าย
  • Test Prioritization: แนะนำ Priority ของ Test Case ที่ควร Execute ก่อน
  • AI-based Defect Prediction: วิเคราะห์ Pattern ของ Defect เพื่อบอกจุดที่เสี่ยงจะเกิด Defect ซ้ำ

Prompt Engineering สำหรับงาน QA

การใช้ Prompt ที่ดีกับ AI ช่วยให้ได้คำตอบที่แม่นยำ รวดเร็ว และตรงกับการทำงานของคุณ โดยมีโครงสร้างที่แนะนำคือ: ระบุเป้าหมาย (Goal), บอกบริบท (Context), ระบุรูปแบบผลลัพธ์ (Output Format), และใส่ตัวอย่าง/ข้อมูลอ้างอิง (Reference)

ตัวอย่าง Prompt ที่ปรับใช้ได้:

  • เขียน Test Script: "ช่วยเขียน Playwright Automation Script ด้วย TypeScript สำหรับเทสการเพิ่มสินค้าลงตะกร้าบนเว็บ โดยใช้ getByRole, locator, และ json เก็บ test data ตามโครงสร้างที่ฉันเคยใช้ พร้อมเพิ่มคอมเมนต์ภาษาไทยในแต่ละขั้นตอน"
  • สร้าง Test Case: "ช่วยสร้าง Regression Test Cases ของฟีเจอร์ [ชื่อฟีเจอร์] ใน Niti Urbanice App โดยเขียนในรูปแบบ Given-When-Then, ระบุ Priority, Test Steps, Test Data, และ Expected Results ด้วย"
  • จัดโครงสร้าง Report: "ช่วยออกแบบ Test Summary Report ที่เหมาะสำหรับสรุปผล Regression Test ใน Google Sheets โดยมีหัวข้อ Test Suite, Total Test Cases, Pass, Fail, Blocked, และ Recommendation ให้เหมาะสำหรับรายงานผู้บริหาร"
  • วาง Test Strategy: "ช่วยวาง Test Strategy สำหรับการทำ Automated Test ของ Living Mart Menu ใน Urbanice App โดยเน้นการ Run บน GitLab CI/CD, รองรับหลาย Browser และทำ Report ผ่าน Allure"
